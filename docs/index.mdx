# STD Plus API

STD Plus is a backend API for managing materials, inventory, orders, and workflow in a glass manufacturing operation.

## Tech Stack

| Technology | Purpose |
|---|---|
| Node.js + Express 5 | HTTP server and routing |
| MongoDB + Mongoose | Database and ODM |
| JWT | Authentication (Bearer Token) |
| Socket.IO | Real-time WebSocket events |
| Zod | Request validation |
| Helmet / CORS | Security |

## Base URL

```
http://localhost:3000
```

All REST API routes are prefixed with `/api`.

## Resources

The API manages the following resources:

| Resource | Description |
|---|---|
| **Workers** | Staff accounts with authentication |
| **Customers** | Client records with discount tracking |
| **Materials** | Material definitions with spec details |
| **Inventories** | Stock levels by material and type |
| **Material Logs** | Audit trail for material movements |
| **Requests** | Customer purchase requests |
| **Withdrawals** | Material withdrawal records |
| **Orders** | Production orders |
| **Claims** | Defect/claim reports tied to orders |
| **Panes** | Individual glass panes through production; laminate groups use **sheet** QRs after merge (**survivor**); list API hides **`merged_into`** unless **`includeMerged=true`** |
| **Pane Logs** | Scan/action history and material timeline for panes (including **qc_pass** / **qc_fail** at QC) |
| **Production Logs** | Action history for every pane at each station |
| **Station Templates** | Configurable UI schemas for station types |
| **Stations** | Work stations in the production line |
| **Notifications** | Real-time alerts for workers |
| **Sticker Templates** | Configurable sticker label layouts |
| **Pricing Settings** | Singleton pricing configuration for glass types |
